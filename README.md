# Arcana

面向中文用户的塔罗牌解读应用。项目采用 **Web 先行、微信小程序随后迁移** 的交付路线。

当前阶段：**P2 跨端工程基础**

## 产品定位

Arcana 提供轻量、沉浸式的塔罗抽牌与牌义解读体验。首版聚焦完成一次完整占卜，不把社区、电商、真人咨询等重运营能力带入 MVP。

> 塔罗内容仅供娱乐与自我探索，不替代医疗、法律、财务或心理咨询。

## MVP

- 今日一牌
- 单牌占卜
- 时间流（过去 / 现在 / 未来）
- 圣三角（现状 / 挑战 / 建议）
- 恋爱关系五牌阵
- 二选一牌阵
- 洗牌、抽牌、翻牌的完整交互
- 正位 / 逆位与对应牌义
- 本地保存最近的占卜记录
- 分享结果卡片
- 必要的隐私说明与免责声明

首版不做：账号体系、支付、社区、真人咨询、复杂牌阵、AI 连续对话。AI 单次解读保留为可插拔能力，是否进入 MVP 由首个原型验证后决定。

## 技术方向

- 跨端框架：Taro
- 前端：React + TypeScript + Vite
- 状态管理：Zustand
- 样式：CSS 变量 + SCSS
- 测试：Vitest；关键流程补充端到端测试
- 数据：首版使用版本化的本地牌库；后端接口按需引入
- 目标平台：H5 / Web、微信小程序

业务规则、牌库数据和页面组件避免直接依赖浏览器 API；平台能力通过适配层统一封装。

## 工程目录

- `packages/tarot-core/`：原型与正式应用共用的牌义、牌阵、牌组素材和纯领域规则。
- `prototype/`：已发布的 Web 交互原型，用于继续做真机体验评审。
- `app/`：正式 Taro 跨端应用，可构建 H5 与微信小程序。
- `docs/`：项目章程、开发计划、架构和持续进度记录。

## 近期决策点

后续开发需要确认：

1. 视觉方向与目标用户画像
2. 微信小程序正式 AppID 与主体信息
3. 首版解读采用固定文案、组合模板还是单次 AI 解读
4. 是否需要手机号 / 微信登录

已确认：Git 仓库名使用 `arcana`；首套牌面采用已审计的原始 RWS TaionWC Pam-A 集合；正式跨端工程使用 Taro 4、React 和 TypeScript。
