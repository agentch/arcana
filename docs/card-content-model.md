# 塔罗牌义内容模型

状态：78 张分层牌义均已 `approved`，运行时抽牌池已扩展为完整 78 张
适用范围：标准 78 张基础牌义、固定模板解读、未来 AI 解读输入

## 1. 目标

正式牌义不能依赖单个超长 `meaning` 字段。每张牌需要同时支持：

- 正位与逆位的基础含义；
- 感情、职场、财务、个人成长等不同问题主题；
- 单牌与多牌阵的不同牌位；
- 摘要、详细解读、象征、行动建议和反思问题的分层展示；
- 经典、东方、赛博等不同视觉牌组复用同一份基础牌义。

牌义数据只表达稳定的领域知识，不保存图片路径、牌背、牌组 ID 或具体牌阵文案。

## 2. 正式 JSON 结构

每张牌使用独立文件，文件名与稳定 `cardId` 一致，例如 `data/cards/major-00.json`。

```json
{
  "schemaVersion": "2.0",
  "contentVersion": "1.0.0",
  "id": "major-00",
  "arcana": "major",
  "number": 0,
  "romanNumeral": "0",
  "name": {
    "zh": "愚人",
    "en": "The Fool"
  },
  "core": {
    "summary": "以信任迈向未知，并保留可调整的风险边界",
    "symbols": ["悬崖", "白犬", "包袱", "太阳"],
    "element": "风"
  },
  "upright": {
    "keywords": ["开始", "自由", "信任", "探索"],
    "overview": "较完整的基础解读……",
    "light": "这张牌积极力量的表现……",
    "shadow": "需要警惕的过度表现……",
    "advice": ["可以采取的行动……"],
    "reflection": "你愿意在哪件事上迈出第一步？"
  },
  "reversed": {
    "keywords": ["冒进", "迟疑", "准备不足"],
    "overview": "较完整的逆位解读……",
    "light": "逆位中仍可利用的力量……",
    "shadow": "潜在盲点与风险……",
    "advice": ["可以采取的行动……"],
    "reflection": "你是在谨慎准备，还是被恐惧困住？"
  },
  "topics": {
    "love": {
      "upright": "感情主题解读……",
      "reversed": "感情主题逆位解读……"
    },
    "career": {
      "upright": "事业主题解读……",
      "reversed": "事业主题逆位解读……"
    },
    "finance": {
      "upright": "财务主题解读……",
      "reversed": "财务主题逆位解读……"
    },
    "growth": {
      "upright": "个人成长解读……",
      "reversed": "个人成长逆位解读……"
    }
  }
}
```

实现 Schema 时，`advice` 使用包含 1–2 项的数组，避免页面再按标点拆分字符串。`symbols` 使用 3–5 项数组。主题 ID 必须来自统一枚举，不允许在单张牌中临时增加同义主题。

Schema v2 使用 `love`、`career`、`family`、`mood`、`finance`、`growth` 六个可用主题。当前问题分类按同名方式显式映射到前四项，`finance` 和 `growth` 作为后续分类预留。映射存放在独立配置中，页面不能自行猜测。

## 3. 内容层级

| 层级 | 字段 | 用途 |
| --- | --- | --- |
| 身份 | `id`、`arcana`、`number`、`name` | 稳定关联与基础展示 |
| 核心 | `core.summary` | 不依赖正逆位和问题主题的牌面核心 |
| 象征 | `core.symbols`、`core.element` | 牌面观察与知识说明 |
| 方位 | `upright`、`reversed` | 正逆位关键词、基础解读、明暗面和行动建议 |
| 主题 | `topics` | 对问题语境进行补充，不复制完整基础牌义 |
| 运行时 | 不落入牌义文件 | 当前牌位、相邻牌、组合摘要和用户问题 |

`light` 与 `shadow` 不是正位和逆位的替代词。每个方位都同时存在可利用的力量与需要留意的风险，避免把逆位简单写成正位的反义。

## 4. 运行时组合

```text
基础牌义
+ 正位或逆位内容
+ 用户问题主题
+ 当前牌阵位置
= 本次单牌解读
```

组合器接收 `cardId`、`orientation`、`topicId` 和 `spreadPosition`，输出结构化的展示模型。牌阵位置只调整解读视角，例如“过去”“挑战”“建议”，不复制或改写整套牌义。

多牌阵在各张单牌解读之后生成消息流摘要，不复述完整问题，也不用「某牌位由某牌回应」的点名句式：

```text
照耀
  过去：主题语境或牌义首句
  现在：主题语境或牌义首句
  未来：主题语境或牌义首句
建议
  现在：现在位行动建议
  未来：未来位行动建议
```

时间流等三牌阵按牌位顺序映射上述结构；完整单牌分层内容进入独立详解页，不在消息流首屏全部铺开。

固定模板必须是默认能力；未来 AI 服务只能基于相同结构增强表达，失败时回退到固定模板。

## 5. 结果页展示规范

### 单牌

- 一句话提示：约 30–50 字；
- 详细解读：约 200–350 字；
- 牌面象征：3–5 项；
- 行动建议：1–2 条；
- 自我探索问题：1 条；
- 单次总阅读量：约 300–500 字。

### 多牌阵

- 消息流先展示牌阵预览与「照耀 / 建议」摘要；
- 照耀覆盖过去发生了什么、现在怎么样、未来会怎么样；
- 建议覆盖现在应该怎么样、未来应该怎么样；
- 完整单牌内容（主题、牌位、象征、明暗面、建议、反思）放在独立详解页；
- 五牌阵优先控制阅读节奏，不把五份完整单牌长文连续堆叠在消息流中。

## 6. 内容质量约束

- 使用可能性、倾向和观察角度，不作绝对预测；
- 不用恐吓、诅咒、保证结果或诱导付费的表达；
- 财务主题只提供自我观察，不给投资结论；
- 心理、医疗、法律和安全问题必须遵守产品边界；
- 正逆位内容需要独立编写，不能机械取反；
- 主题内容用于补充语境，不能与基础牌义相互矛盾；
- 每张牌记录内容版本、审核状态和最后审核时间。
